java基础

2014-06-17
java

1.java初始化顺序先，类成员变量，再构造函数创建对象。有限原则，静态优先非静态，父类优先于子类。按成员变量的定义顺序来初始化。
2.基本类型是值传递，引用类型引用传递。引用类型只能通过clone(),来返回对象的复制，实现cloneable。override clone()方法。spuer.clone().这样才能把浅复制引入。
当对象中存在非基本数据类型时候，需要深度复制。对象的非基本类型数据也要复制。浅复制没有吧对象中的非基本类型数据给复制了，还是原来的引用数据。
3.Class.forName(“Test”) Test.class test.getClass(),所以java可以通过，new 返射 clone 反序列化 4种方式来创建对象。
4.不要单纯的为了代码复用实现继承，会破坏可维护性，采用接口加组合有更好的扩展性。类的方法才有多态，成员变量没有多态。Base.i 不会是实现类的i.
5.包含抽象方法的类就是抽象类，接口，也就支持抽象类定义的两种机制。接口和抽象类都不能被实例化。
6.内部类可以看成是外部类的一个成员。可以调用外部类的private.静态内部类，可以直接实例化，只能调用外部类的静态方法，非静态不能直接实例化，但外部类的都可以调用。
匿名内部类 不能有构造函数和静态成员，一定在new 后面。构造器中的super()必须在第一条。
7.contine< break < return break 想跳出多重循环， hear: break hear;在循环外加一个标识符。
8.final 不可变修饰，引用都不可变。final String s = “test” s =”test2”,会编译出错。String 就是final class，不能被继承。finally。 配合try{}catch(){}finally{} 来使用。finalize 建议垃圾回收。
9.volatile,提高可见性，变量不在冲缓存中取，而是每次都在内存中取。每次让别的线程在寄存器中的变量失效。instanceof 二元运算符，判断对象是否是一个类，或接口。
10。strictfp strict float point 关键字修饰,所有浮点型计算都是精确的。BigDecimal 一样的效果。超过16位有效位。处理时候。

  1. 8种基本数据类型 byte（-128-127 第一位是正负标识）,short,int,long,float,double,char,boolean. 封装类 Byte,Short,Integer,Long,Float,Double,Character,Boolean. byte<short<char<int<long<float<double. 小到大 字段转换。boolean 不能自动转换。
  2. += 为规定运算符。 short s1 =1 ;s1 =s1+1 报错，但 s1+=1;就可以。 >>> 有符号右移，也就是标识符最前面的1位，也跟着移动，负数会移成正数。
  3. && 和& 前面是短路与，|| 和| 前面是短路或。 int i = 5 i++ =5 先调用在加, ++i =7 先加再调用.

  4. << 移动大于可移动位数是 只能移动取摸，也就是int 33%32=1, long 65%64=1.

  5. char =2byte 中文2字节，因为1字节，所以可以用来判断中文。
    16.数组是对象 可以用instanceof 来判断。数组的大小不能随意改变，初始化的时候一定确定。
    17.length 是数组的属性，length()是String 的长度等。
    18.serialVersionUID 不同的话，反系列化就会抛异常。invalidClassException.SerialVersionUID,不写，序列化时候就要生产。增加兼容性。
    19。classLoad 实质也是类，是吧类文件从硬盘读取到内存中，隐式加载 new ,显示加载 Class.forName();需要的类后续才会加载，这样节省内存。 .class是可以被动态加载的单元。BootStrap Loader负责加载系统类。ExtClassLoader 负责加载扩展类。 AppClassLoader 负责加载应用类。类加载是双亲委托模型，也就是都从父类加载器开始加载，这样才不会出现用户自己写个String类，把系统的String给盖了。如果不是加载path下面的jar,需要自定义ClassLoad.继承ClassLoad.重写findClass.tomcat 有两个自动以的加载器一个standard 和webApp. URLClassLoader 来加载的plugin.
    20.stack 线程独立的，方法执行的时候有栈帧来操作出入。栈中的引用变量来访问堆中的对象。Object s = new Object();前面在栈中，后面再堆中。
    21.hashMap (capacity load factor) threshold 没有超过这个数的话不会resize()。默认是16 1<<4 capacity,threshold(160.75=12)ConcurrentHashMap,Segment 加锁，ReentrantLock,第一次hash segment,后一次hashEntry 分化加锁。效率更高。 HashMap table ->红黑树。
    22.ArrayList 扩展就为1.5倍。Vector就为2倍。LinkedList 链表，访问效率低，但插入效率高。非线程安全。
    23.同步在资源访问时候，是为了让支援安全，也就是synchronized.同步机制。上锁。
    24.异步Callable,返回Future. future.get()时候回当前线程就会堵塞。直接调用run()就是方法调用就是同步的了。 ThreadPoolExecutor ExecutorService 执行runable callable 返回future.线程池实现的就是。提交runable,创建thread.并且执行。ThreadPool里执行work.sleep是thread,方法是暂停一段时间交出cpu,但不会释放锁。对象还在他手上，wait()是对象方法，释放占有的锁，并等待,wati()要放在同步块中，直到notify.sleep 不释放锁，所以容易产生，死锁，所以不推荐使用。yield()只是放出执行权，但又可能会重新获得。
    25.synchronized 使用的是Object对象自身的notify,wait,notityAll调度机制。Lock使用的是condition.Lock 使用的范围小，但控制性比较强。功能强大，需要手动在finally里面手动释放锁。synchronized的同步范围是在定义的方法体中。静态sychronized和非同步方法，都不会被同步方法影响。
    26.守护线程t.setDaemon(true),在start()之前。没有非守护线程后，守护线程也会立刻终止。垃圾回收期就是典型的守护线程。
    27.jion()现实同步， t.join(10) 阻塞等待线程执行。
    29.GET获取服务端资源，资源在url中，所以不安全，POST修改服务端资源。请求体附件中，安全的。
    30.Servelet 来处理用户的web请求，相应的url对应相应的servlet.创建request和response来调用相应的方法。组装http格式返回给客户端。servlet容器启动的时候就会被初始化，每次请求过来都会创建新线程来调用server()根据请求的method来调用doGET() OR doPOST();
    31.cookie和session的区别，一个是浏览器端的小文件，一个是服务器端的保证状态的。SessionID使用cookie来传递的，所以禁用了cookie ,session也就用不了了。当也可以吧sessionid放到URL中.
