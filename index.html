分布式ID生成器

2016-11-24
分布式

生成要求：尽量短、无规则、无顺序。 自己理解为什么

目前我正在写的 ID 生成服务的基本思路是这样的：

  1. ID 生成服务管理多个不同名字的 ID 池 (pool)。
  2. 每种不同类型的 ID 属于不同的 ID 池，比比如用户 ID 池、订单 ID 池、商品 ID 池等等。
  3. 每个 ID 池由多个定长 ID 块 (block) 构成。
  4. 每个 ID 块包含一段连续的 ID，比如第一块是从 0~1023，第二块从 1024~2047，以此类推。
  5. 每个 ID 块并不完全分配，而是按照一个给定的填充率 (fill ratio) 随机选择来分配，比如假设填充率是 50%，那么每个 ID 块中只有大约一半的 ID 会被分配。也就是说每个 ID 块是有随机空洞的。
  6. 如果某个 ID 块中可用 ID 被分配完毕，服务会自动生成下一个新的 ID 块，并按照填充率去掉不可用 ID。生成新的 ID 块时需要记录下最后一个 ID 块的起始 ID。
  7. 已经分配过的 ID 会写入一个 append-only 日志。新的 ID 块生成时会创建一个新的 append-only 日志（因为旧的已经不需要了）。
  8. 服务重新启动的时候先拿到最后一个 ID 块的起始 ID，再读取 append-only 日志将已经分配过的 ID 剔除，再剔除掉部分未分配的 ID 以维持填充率。
  9. 客户端发送请求访问 ID 生成服务。服务端返回 ID 做为回复。

优点是逻辑简单，生成的 ID 短小，单个进程维护起来方便。缺点是知道生成的 ID 的上限（可以用很大的块部分解决这个问题）。大概这样。欢迎讨论。

异步化方案

2016-11-12
java

异步化的核心，减少线程等待io,充分利用线程资源，使用callback来处理。

1.纤程，线程是由os来调度的，而纤程的调度是应用层面实现的，只维护执行栈信息，不分配资源，所以能建立成千上万个的纤程。quasar是JVM上实现的纤程库。
2.actor模型，传统并发编程，主要是内存共享的方式来实现多线程之间的通信，而actor只要是用于事件驱动的方式来进行通信。是一种分离，减少io等待。共享内存有锁，就有等待。
3.RX异步编程。函数式异步化封装。

简单根据servlet 3.0 与spring 3.2 来实现异步化。
ListeableFuture.addListner 注册监听事件来完成最后的返回。这都需要Servlet 和springmvc的支持。
注意事项
1.tomcat7才支持servlet3.0
2.配置web.xml dispatcher 和所有filter/ture
3.ThreadLocal中的变量，可能会因为异步化，照成不在同一个线程中，要妥善处理。
4.异常处理，异步化异常封装在ExecutionException中，同步代码中的catch 要捕获的是ExecutionException,并且e.getCause()来获取异常。这一步的异常封装和抛出主要有应用来实现了。

metaspace

2016-11-07
JVM

Metaspace

jdk8中没有Prem，Perm的取消主要是因为动态类加载越来越多，Perm的大小已经不太可控。为让整个系统可控性增加，所以在jdk8中引入了MetaSpace来替代。

metaSpace 也有厨房Kless 信息的的区域，也就是class文件在jvm里的运行时数据结构。不过A.class其实是存在heap里的。这里也会有一个大内存32G情况下关闭指针压缩开关。
metaspace除了厨房Kless以外，还会存放NoKless。比如method,constanPool等。这块虽然叫NoKless，但也可能会存放Kless信息，比如指针压缩关闭的情况下。因为NoKlass，是由一块块内存慢慢组合起来的，在没有达到限制条件的情况下，会不断加长这条链，让他继续工作。

红黑树

2016-11-05
数据结构

###红黑树
红黑树与avl是二叉树的延伸，主要目的是降低二叉树搜索时候，应为不平衡造成的时间复杂度升高。在平衡的状态下一棵基本平衡的二叉树的搜索时间为O(lgN);
所以在时候二叉树查询时候，二叉树的平衡与否，直接影响搜索时间的大小。

LRU 算法

2016-10-17
lucene

LRU， Least Recently Used

  1. 新数据插入到链表头部；
  2. 每当缓存命中（即缓存数据被访问），则将数据移到链表头部；
  3. 当链表满的时候，将链表尾部的数据丢弃。

【命中率】
当存在热点数据时，LRU的效率很好，但偶发性的、周期性的批量操作会导致LRU命中率急剧下降，缓存污染情况比较严重。
【复杂度】
实现简单。
【代价】
命中时需要遍历链表，找到命中的数据块索引，然后需要将数据移到头部。
