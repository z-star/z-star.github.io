Linux I/O模型
1.阻塞模型：recvfrom之后一直等待数据，直到数据准备好，复制到用户空间之后才返回。
2.非阻塞模型：的recvfrom之后一直轮询，直到数据到来。复制到用户空间返回。
3.I/O复用模型：select/poll epoll事件选择器扫描，等待rollback。
4.信号驱动I/O模型：
5.异步I/O

###NIO
1.缓冲区Buffer
新NIO中写入或者读出是到Buffer，而不是直接到stream中。可以理解为一个字节数组(ByteBuffer).每一种基本类型都会对应一种缓冲区除了boolean。
2.Channel
通道与stream不同，通道是双向的既能读又能写。而InputStream,OutPutStream看命名就知道非双向。
Channel子类中大概分两类一类网络读取，一类文件读取。一个多路选择器selector可以轮询注册在上门的channel的状态。

Centos7安装MySQL5.7.9
介绍：MySQL的安装方法有多种，如yum安装，rpm包安装，rpm源码编译安装，源码编 译安装，通用预编译包安装。其中yum安装和rpm包安装比较简单，软件安装时会自动默 认设置，但可定制性弱版也较旧。源码包编译安装可定制性强版本也新。下面是通用预编译 包安装和源码编译安装的方法，MySQL的版本为最新的5.7.9版。在MySQL5.7.9中有一些新 的特性如：
1、 初始化数据库时不再使用”–mysql_install_db”而是使用了 “–initialize”或 是”–initialize-insecure 〃，区别在于”–initialize”会生成一个随机密码而”–initialize-insecure”不会
生成密码。
2、 不在生成.mysql_secret文件；
3、 不再创建匿名用户MySQL账户；
4、 不再创建测试数据库。
编译源码方式安装MySQL数据库
1、安装文件准备
.下载cmake-3.2.2.tar.gz …新版的MySQL用CMAKE编译安装
http://www.cmake.org/download/
.下载 ncurses-6.0.tar.gz …安装 MySQL 时需要
ftp://ftp.gnu.org/gnu/ncurses/
.下载 bison-3.0.4.tar.gz …安装 MySQL 时需要
http://ftp.gnu.org/gnu/bison/

分布式链路跟踪调度系统

2015-07-12
tools

面对日益复杂的大规模分布式集群环境，入口流量涨了，不知道要升级扩容哪些后端服务；数据库撑不住了，不知道是哪些应用读写多了；订单不见了，付款成功出不了票了，grep n多台机器日志，联系n个运营团队也没找到答案；这些问题都可以交给分布式链路跟踪调度系统来解决。

分布式链路跟踪调度系统源于Google Dapper论文，继Twitter的ZipKin，eBay的Centralized Activity Logging (CAL)，淘宝的鹰眼，京东商城的Hydra之后，它的强大之处在于能做到追踪每个请求的完整调用链路，收集调用链路上每个服务的性能数据，并与异常监控集成，在收到报警时能从监控峰值点关联上若干异常trace样本，快速定位问题。相信在不久的将来，它能通过计算和比对性能数据，分析系统瓶颈点，并最终反馈到服务治理中。
