elasticsearch 待学习

elasticsearch 源码和totem类似，都是目前比较流行的module,模块装配，但因为guice，的引入，代码显得更加的简洁。
所有的都会按服务切分，一个分布式搜索引擎的业务切分模块。
主函数启动时都会初始化所有的服务。setting,search,index,agg,cluster.等。

###Module分析
CacheRecyclerModule-CacheRecycler 缓存服务。
PageCacheRecyclerModule-MockPageCacheRecycler 分页缓存服务
CircuitBreakerModule-HierarchyCircuitBreakerService 熔断模块，阻止内存占用超过等（indices.breaker.total.limit：70%）
在内存使用中fileddata比较多。也比较复杂。
ChildMemoryCircuitBreaker.addEstimateBytesAndMaybeBreak() 判断是否超过限制。PagedBytesIndexFieldData.loadDirect(加载fielddata时候判断，内存存储实例 OrdinalsStore。IndicesFieldDataCache.load()
NodeSettingsService –节点动态更新配置的实现类.
BigArraysModule-BigArrays
NodeModule–(InternalNode,NodeSettingsService 配置管理 ,NodeService 节点服务NodeStats，NodeInfo的Response)
NetworkModule–NetworkService() ip port 服务。
ScriptModule-NativeScriptEngineService 解析各种script请求。主要用于search.
EnvironmentModule-Environment环境文件
NodeEnvironmentModule-NodeEnvironment节点环境文件索引级别。
ClusterNameModule–ClusterName 集群名字(elasticsearch)
ThreadPoolModule–ThreadPool 线程总管理
DiscoveryModule–ElectMasterService(选举服务)LocalDiscovery(本地发现)ZenDiscovery(非本地发现)UnicastZenPing(ping服务)Discovery DiscoveryService
ClusterModule–DynamicSettings(配置动态变更)cluster**大模块
RestModule–RestController注册 filters RestRequest ,registerHandler注册action，RestActionModule。
TransportModule–NettyTransport(netty ServerBootstraps初始化）dispatcher为MessageChannelHandler，分发action.
HttpServerModule–NettyHttpServerTransport
IndicesModule–核心模块，分三块（registerBuiltinQueryParsers，registerBuiltInMappers，registerBuiltInMetadataMappers）改写FunctionScoreQueryParser来实现打分.IndicesLifecycle(所以周期生命监听处理)IndicesService(所以创建删除等操作)IndicesStore（所以存储）IndicesClusterStateService（）IndexingMemoryController（更新所有ShardBuffers）IndicesQueryCache(lucence中的LRUQueryCache)
IndicesFieldDataCache()
SearchModule–核心模块

1.IndexReader打开索引文件，读取并打开指向索引文件的流。
2.用户输入查询语句
3.将查询语句转换为查询对象Query对象树
4.构造Weight对象树，用于计算词的权重Term Weight，也即计算打分公式中与仅与搜索语句相关与文档无关的部分(红色部分)。
5.构造Scorer对象树，用于计算打分(TermScorer.score())。
6.在构造Scorer对象树的过程中，其叶子节点的TermScorer会将词典和倒排表从索引中读出来。
7.构造SumScorer对象树，其是为了方便合并倒排表对Scorer对象树的从新组织，它的叶子节点仍为TermScorer，包含词典和倒排表。此步将倒排表合并后得到结果文档集，并对结果文档计算打分公式中的蓝色部分。打分公式中的求和符合，并非简单的相加，而是根据子查询倒排表的合并方式(与或非)来对子查询的打分求和，计算出父查询的打分。
8.将收集的结果集合及打分返回给用户

refresh:刷到可搜索。
flush：内存刷到segment文件上。

netty

####BIO(同步阻塞式)

####NIO（异步非阻塞式）
Linux I/O模型
1.阻塞模型：recvfrom之后一直等待数据，直到数据准备好，复制到用户空间之后才返回。
2.非阻塞模型：的recvfrom之后一直轮询，直到数据到来。复制到用户空间返回。
3.I/O复用模型：select/poll epoll事件选择器扫描，等待rollback。
4.信号驱动I/O模型：
5.异步I/O

###NIO
1.缓冲区Buffer
新NIO中写入或者读出是到Buffer，而不是直接到stream中。可以理解为一个字节数组(ByteBuffer).每一种基本类型都会对应一种缓冲区除了boolean。
2.Channel
通道与stream不同，通道是双向的既能读又能写。而InputStream,OutPutStream看命名就知道非双向。
Channel子类中大概分两类一类网络读取，一类文件读取。一个多路选择器selector可以轮询注册在上门的channel的状态。

hashcode

2015-10-12
java

＊ hashcpde

＊hashcode
关系到对象的索引位置，对象的hash-relevant改变，不会重新计算hashcode,所以可变对象会有hash 冲突。
结论：所以最好不要使用可变字段来计算hashcode.

  • hashcode计算时候选择31作为乘积，是因为31是素数，减少冲突，而且位数并不大，不会发生溢出。
  • 如果重写equals就 尽量重写hashcode,防止冲撞。
