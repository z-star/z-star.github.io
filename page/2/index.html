Observer 观察者模式

2014-04-26
design pattern

#Observer 观察者模式
＊注册 撤销 需要通知的用户。本质是一个消息发布。观察者顾名思义，就是每个观察人员查看变化及时通知。 Observer subject
*优点：作为一个动作的连接点，能实现通知功能，pub／sub
｀｀｀javascript
public interface Observer{
public void notify();
}

public class OneObserver implments Observer{

  @Override
  public void notify(){
  system.out.print("one notify");
}

}

 public class TwoObserver implments Observer{

 @Override
 public void notify(){
  system.out.print("Two notify");
}

}

public interface Subject(){

public void addObserver(String obName,Observer ob);

public void removeObserver(String obName);

public void notifyObserver();

}

public class OneObserver implements Subject{

private Map<String,Observer> map = new HashBiMap.create(); ;

public void addObserver(String obName,Observer ob){
  if(!map.contains(obName))return;
    this.map.put(obName,obj);
}

public void removeObserver(String obName){
    tap.map.remove(obName);
}

public void notifyObserver(){
  for(String key:map.keySet()){
    Observer obj = map.get(key);
    obj.notify();
  }
}
public static void main(String[] args){
  Subject sub = new OneObserver();
  sub.add(new OneObserver());
  sub.add(new TwoObserver());
  sub.notifyObserver();
}

}

strategy 策略模式

2014-04-26
design pattern

#strategy 策略模式
＊将各种能拆分，复用的组建或者行为，抽象出来，并且封装起来。比如一个厨房水槽实例需要装一个水龙头，水龙头的具体实现是独立的。需要什么类型的水流形状，需要引入不同类型的龙头。在进行实现主体类时。组装各种需要的组建。让组建独立于使用者。
*优点：相比继承，碎片化，模块化，复用性，更高。
｀｀｀javascript
public interface Tap{
public void open();
}

public class OneTap implments Tap{

@Override
public void open(){
  system.out.print("one top")
}

}

 public class TwoTap implments Tap{

@Override
public void open(){
  system.out.print("Two top")
}

}

public class Strategy{

private Tap tap ;

public void setTap(Tap tap){
    this.tap =tap;
}

public void open(){
    tap.open();
}
}
