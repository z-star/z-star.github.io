Linux I/O模型
1.阻塞模型：recvfrom之后一直等待数据，直到数据准备好，复制到用户空间之后才返回。
2.非阻塞模型：的recvfrom之后一直轮询，直到数据到来。复制到用户空间返回。
3.I/O复用模型：select/poll epoll事件选择器扫描，等待rollback。
4.信号驱动I/O模型：
5.异步I/O

###NIO
1.缓冲区Buffer
新NIO中写入或者读出是到Buffer，而不是直接到stream中。可以理解为一个字节数组(ByteBuffer).每一种基本类型都会对应一种缓冲区除了boolean。
2.Channel
通道与stream不同，通道是双向的既能读又能写。而InputStream,OutPutStream看命名就知道非双向。
Channel子类中大概分两类一类网络读取，一类文件读取。一个多路选择器selector可以轮询注册在上门的channel的状态。

＊hashcode
关系到对象的索引位置，对象的hash-relevant改变，不会重新计算hashcode,所以可变对象会有hash 冲突。
结论：所以最好不要使用可变字段来计算hashcode.

  • hashcode计算时候选择31作为乘积，是因为31是素数，减少冲突，而且位数并不大，不会发生溢出。
  • 如果重写equals就 尽量重写hashcode,防止冲撞。

进程和线程

2015-10-12
java

1.为什么要有进程

进程是为了解决，cpu的使用效率，让系统能够并发操作。这样才能在单台计算机，丰富的体验。因为进程分割内存，并交替使用cpu来运算。才能做到cpu使用率提高。在程序IO的情况下，cpu是闲置的，io时间越长闲置时间越长，多个程序并发操作。切换非常快，人根本没法感知，所以才能让如今的操作系统有很好的体验。

2.为什么要有线程
在出现了进程之后，操作系统的性能得到了大大的提升。虽然进程的出现解决了操作系统的并发问题，但是人们仍然不满足，人们逐渐对实时性有了要求。
换句话说，进程让操作系统的并发性成为可能，而线程让进程的内部并发成为可能。但是要注意，一个进程虽然包括多个线程，但是这些线程是共同享有进程占有的资源和地址空间的。进程是操作系统进行资源分配的基本单位，而线程是操作系统进行调度的基本单位。

3.多线程并发

　　由于多个线程是共同占有所属进程的资源和地址空间的，那么就会存在一个问题：如果多个线程要同时访问某个资源，怎么处理？线程安全问题就这样产生。

介绍：MySQL的安装方法有多种，如yum安装，rpm包安装，rpm源码编译安装，源码编 译安装，通用预编译包安装。其中yum安装和rpm包安装比较简单，软件安装时会自动默 认设置，但可定制性弱版也较旧。源码包编译安装可定制性强版本也新。下面是通用预编译 包安装和源码编译安装的方法，MySQL的版本为最新的5.7.9版。在MySQL5.7.9中有一些新 的特性如：

生成密码。
2、 不在生成.mysql_secret文件；

解决:
/etc/init.d/mysql stop …停止 mysql 服务
mysqld_safe -skip-grant-tables -skip-networking & …跳过权限表控制，跳过 TCP/IP 协议在本机访问
mysql -u root -p mysql —提示输入密码时直接到7丨（

mysql> flush privileges; —刷新MySQL的系统权限相关表使其生效

#/etc/init.d/mysqld restart …重启 mysql 服务

systemctl start mysql

Starting MySQL.. ERROR! The server quit without updating PID file (/usr/local/mysql/data/localhost.localdomain.pid).
