elasticsearch 待学习

elasticsearch 源码和totem类似，都是目前比较流行的module,模块装配，但因为guice，的引入，代码显得更加的简洁。
所有的都会按服务切分，一个分布式搜索引擎的业务切分模块。
主函数启动时都会初始化所有的服务。setting,search,index,agg,cluster.等。

InternalNode 节点

###Module分析
CacheRecyclerModule-CacheRecycler 缓存服务。concurrencyLevel 类似concurrentMap中的级别服务。Delegate 给某一个concurrent。
默认SOFT_CONCURRENT,也就是软应用。

PageCacheRecyclerModule-MockPageCacheRecycler 分页缓存服务
PageCacheRecycler 1.7默认情况下，默认的模式是CONCURRENT
limit默认10%heap pageSize 默认 16KB。maxPageCount=limit/pageSize; maxBytePageCount占比。

CircuitBreakerModule-HierarchyCircuitBreakerService 熔断模块，阻止内存占用超过等（indices.breaker.total.limit：70%）
在内存使用中fileddata比较多。也比较复杂。 overhead 1.03,
ChildMemoryCircuitBreaker 主要的方法类，addEstimateBytesAndMaybeBreak判断是否过载。
ChildMemoryCircuitBreaker.addEstimateBytesAndMaybeBreak() 判断是否超过限制。PagedBytesIndexFieldData.loadDirect(加载fielddata时候判断，内存存储实例 OrdinalsStore。IndicesFieldDataCache.load()
BigArraysModule-BigArrays ，pageSize 16k/占用bit. 当要申请的空间大于pageSize的时候就需要使用到，bigArray了。bigArray使用的也是二维数组结构。底层都是Recycler。
PluginsService 两种plugin jvmPlugins 和 sitePlugins

IndicesModule–
OpenBitSet–使用的重要数据结构
OpenBitSet的原理

二进制下标表示数字所以得到
[0,63]之间的数 需要1个long类型串联起来
[0,127]之前的数，需要2个long串联起来
[0,64N-1]需要N个long串联起来。
所以OpenBisSet的核心就是long类型的数组bits.
long index = 64*N+m 其中 N就是在long数组中的坐标。m表示bit[N]的这个long中二进制的第m位。
set方法就4个操作，第一个
int wordNum = (int)(index >> 6); //计算出坐标所在的long数组的位置。
long bitmask = 1L << (int)index; //计算出移动该坐标后的long的大小
this.bits[wordNum] |= bitmask; //和所在坐标的long进行或运算，也就是并集。

最后总结一下OpenBitSet数据存储的特点：OpenBitSet无法存储重复的数据。数据存储到OpenBitSet中后就是有序的了。OpenBitSet适合存储密集程度高，且量大的数据。OpenBitSet中存储的数据适合位运算，比如取交集、并集、补集
核心模块，分三块（registerBuiltinQueryParsers，registerBuiltInMappers，registerBuiltInMetadataMappers）改写FunctionScoreQueryParser来实现打分.IndicesLifecycle(所以索引周期监听处理)
IndexShard 创建每一个shard对应一个实例。在shards里被引用。
indexService 创建每一个index对应一个实例。在indices里被引用。
IndicesService(所以创建删除等操作)是indexService的操作类。
RecoveryTarget recovery的具体操作类。startRecovery,获取恢复的node,把分片数据发送过去。doRecovery 获取metadataSnapshot.InternalClusterService updateTask 检测状态，来做recovery的处理。
IndicesStore（所以存储）
IndicesFilterCache(WeightedFilterCache 来存储,weight回收, 内存里存的DocISet,FixedBitSet)
IndicesFieldDataCache （load来加载fieldData缓存数据）
IndicesClusterStateService()
IndexingMemoryController（更新所有ShardBuffers）IndicesQueryCache(lucence中的LRUQueryCache)

SearchModule–核心模块
SearchServiceTransportAction 注册。
search 过程。入口主要是SearchServiceTransportAction 完成一个shardTarget
TransportActionNodeProxy.execute.搜索遍历所有的节点。
TransportSearchAction 为入口，来区分不同的searchType.
TransportSearchQueryThenFetchAction

refresh:刷到可搜索。
flush：内存刷到segment文件上。

action 请求端的Action

searchAction为例子。先发送给TransportSearchAction，再由TransportSearchAction 中的registerRequestHandler 来注册，并用 TransportHandler 来分发，确定具体的search类型再发送给别的action。

消息的listener 还是依赖RequestHolder对象来存放listener对象，以requestId为key.每次收到消息都来处理response对应的requetId。
